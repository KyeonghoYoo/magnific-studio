# 파일럿 1 회고 — riderly-biker-wave (2026-07-12)

첫 실전 파일럿: Xposz 트렌드 리서치 → 기획 → 캐릭터 → 콘티 → 생성 → 후반, 총 생성 비용 **0크레딧**(GUI unlimited). 최종: 1080×1920 · 9.42s · −14 LUFS. 사용자 평가: "AI Slop 같은 게 나올 줄 알았는데 꽤 괜찮네. 100% 만족은 아니지만 작동한다."

## 알려진 결함 (다음 이터레이션 1순위)

1. **VO 말미 잘림** — "라이덜리"의 "덜리"가 잘림. 원인: VO를 4.9~9.7s에 배치했으나 dip_to_white(600ms) 겹침이 총 길이를 10.0→9.42s로 당기는 것을 미반영. **규칙화 필요: VO/오디오 배치는 전환 겹침 반영 후의 실효 타임라인 길이 기준으로 산술** (렌더러의 offset_k 산식과 동일 원리 — 오디오에도 적용). 즉효 수정: start_sec 4.5 또는 엔드카드 +0.5s.
2. **엔드카드 연출 재검토** (사용자 디렉팅 대안 2안):
   - (a) 스톡 하늘 대신 **순백 배경** + 로고·카피 — 미니멀, 브랜드 오렌지가 더 튐
   - (b) **라이더들이 웃으며 화합하는 장면 → 블러 전환**되며 카피·로고 등장 — 감정 페이오프를 시각으로 완결(현재는 인물이 빠진 하늘이라 감정 연결 약함). 콘티에 화합 숏 1개 추가 필요(생성 1회)
3. sh1 훅의 실드 클릭 임팩트가 미세 — "탁" 순간이 약함. 트림 구간 재선별 또는 클릭 SFX 강조.

## 검증된 것 (지키면 되는 것)

- **포지션 락 = 단일 클립**: 동일 포지션 연속 컷을 8s 단일 클립로 생성 → 후반 트림. 카메라 리그 튐 원천 소멸 — 이번 파일럿 최대 품질 기여.
- **환경 플레이트 3중 잠금**(조명 문장+팔레트+locations 플레이트): 배경 발산 해결.
- **네이티브 SFX 하이브리드**: 엔진·도플러는 화면 동기가 공짜 — 스톡 타이밍 매칭보다 우월. 음악·VO만 후반 스택.
- **음악 빌드 실측 정렬**(RMS 2s창): 상승부를 감정 클라이맥스에 — librosa 없이 ffmpeg volumedetect로 충분.
- **실물 자산 합성**(로고 flood-fill 투명화 + Pillow 오버레이): drawtext 없는 ffmpeg 빌드에서도 세이프존 정확 번인.
- **user-run(GUI unlimited) 모드**: 에이전트=그래프·프롬프트·검증, 사용자=재생 버튼 — 파일럿 전체 0크레딧.
